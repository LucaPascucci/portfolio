C#

Linguaggio compilato JIT (Just In Time) durante l'esecuzione del programma.
  1. Prima fase (compile time): Complilazione intermedia del codice (Fatta dal framework .Net) che genera il CIL (Common Intermediate Language)
  2. Seconda fase (runtime): Traduzione del CIL in CLR (Common Language Runtime)
Garbage Collector presente!!

Paradigmi

  • Programmazione Strutturata:
    • Selezione (IF statement)
    • Iterazione (Cicli FOR)
  • Programmazione Imperativa: programma come insieme di istruzioni ordinate (ad esempio codice all'interno di una funzione)
  • Programmazione Object-Oriented
    • Classe -> metodi + attributi (variabili e costanti)
    • Oggetto -> instanza della classe
    • Incapsulamento: Black-box -> Stato della classe (variabili) accessibili solo tramite metodi
    • Ereditariet√†: Gerarchia di classi e interfacce
  • Programmazione Funzionale: non esiste il concetto di stato esistono solo funzioni (-> Input -> Funzione -> Output). Si utilizza molto la ricorsione

Astrazioni

  • Classi (astratte)
  • Interfacce
  • Delegate (Riferimento ad un metodo/oggetto)
  • Eccezioni

Organizzazione

  1. Namespace
  2. Classi
  3. Metodi

Keywords

  • get / set : per creare velocemente getter e setter di una variabile
  • as : converte, in modo esplicito, una variabile in un determinato tipo
  • base : per utilizzare i metodi della classe padre
  • checked : serve per controllare se una variabile va in overflow
  • continue : fa continuare il ciclo for senza eseguire il codice sottostante che in pratica viene saltato
  • delegate : Firma di un metodo che permette l'incapsulamento di un metodo anche anonimo e consente il passsaggio di metodi come se fossero parametri
